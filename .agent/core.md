# Core Loop

1. Resolve the oldest open interrupt.
2. Otherwise resume the next incomplete plan step.
3. Before editing, name the active step and acceptance criteria.
4. If pausing, record `Pauses:` and `Resume From:`.
5. Log issues immediately and update lessons immediately after.
6. Verify acceptance before marking work complete.
7. Update required records before moving on.
8. Report assumptions and decisions to the user after each completed step or interrupt.
9. Stop and surface conflicts or ambiguity.
